===============================================================================
#                               INSTALLATION                                  #
===============================================================================
This utility depends on ODBC drivers and therefore depends on either native
Microsoft ODBC drivers or unixODBC for non-Windows.  All dependencies for
non-Windows have been included in this package.  In addition, the datasource
must be configured in the TNSNAMES.ora file under $ORACLE_HOME/network/admin.
This utility also requires that the Oracle dump is generated by expd and not
exp.
===============================================================================


===============================================================================
#                               DESCRIPTION                                   #
===============================================================================
This utility is meant to help with Oracle database refreshes, be it prod-to-dev
or dev-to-prod, etc.  There are 2 paths that can be taken:
1) Drop the current schema (if it exists)
2) Create the base schema
3) Import data using impdp

OR

1) Overwrite current data in the schema through a replace of the tables.
===============================================================================


===============================================================================
#                                  FLAGS                                      #
===============================================================================
* -v: Verbose mode
* -d: Name of the data source (must be in TNSNAMES.ORA)
* -s: Name of the user schema (database)
* -p: Password to use for the schema
* -P: Password for the Oracle system user
* -f: Path to the dump file to import
* -i: Oracle directory name
* -D: Drop and re-recreate the schema
